# SQL-Data-Analytics-Pizza-Project
🍕 Just wrapped up a deep-dive SQL project: Pizza Sales Business Analysis!
Pizza Sales Data Analysis (SQL)

## 📌 Project Overview
This project focuses on analyzing business data for a pizza restaurant chain using **MySQL**. The goal is to transform raw data into meaningful insights regarding sales performance, customer behavior, and product popularity to support data-driven decision-making.

The analysis is structured into three levels: **Basic**, **Intermediate**, and **Advanced**.

## 🛠️ Tools Used
* **Database:** MySQL
* **Language:** SQL
* **Concepts:** Joins, Subqueries, CTEs, Window Functions, Aggregate Functions.

## 📊 Key Business Questions Answered
### 1. Sales Performance
* Calculated the **total revenue** ($817,860.05) and total number of orders (21,350).
* Identified the **highest-priced pizza** (The Greek Pizza at $35.95).
* Analyzed **cumulative revenue** generated over time to track growth trends.

### 2. Product Analysis
* Determined the **top 5 most ordered pizza types** by quantity (led by The Classic Deluxe Pizza).
* Identified the **most common pizza size** (Large).
* Found the **category-wise distribution** of pizzas (Classic, Supreme, Veggie, Chicken).

### 3. Consumer Insights
* Analyzed the **hourly distribution of orders** to identify peak hours (highest activity at 12 PM and 1 PM).
* Grouped orders by date to calculate the **average number of pizzas ordered per day** (138 pizzas).

## 🚀 SQL Techniques Demonstrated
* **Data Retrieval & Filtering:** Using `SELECT`, `WHERE`, and `LIMIT`.
* **Joins:** Combining `orders`, `pizzas`, and `pizza_types` to create a unified view.
* **Aggregations:** Using `COUNT`, `SUM`, `AVG`, and `ROUND`.
* **Advanced Analytics:** Implementing **Window Functions** (`SUM OVER`) for cumulative totals and subqueries for complex logic.
